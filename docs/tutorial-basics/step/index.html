步骤

步骤由执行条件,执行动作,结束条件组成,是用户在某场景下操作手机的抽象.

步骤分为五个状态:开始,执行,完成,结束,超时,如下图所示

基本信息​

步骤的基本信息包括:名称,重复次数,是否可选,间隔时间,超时时间.

  1. 名称即步骤的名称,在一个流程中,步骤名称必须唯一.
  2. 重复次数即重复执行的次数,每一次执行都会匹配步骤执行条件,只有在符合执行条件的情况下,步骤才会重复执行,达到设定的次数后,步骤才会进入完成状态.
  3. 是否可选表示如果执行条件没有满足的情况下,是否继续等待,直到满足执行条件或者超时.如果步骤可选,第一次检测到执行条件不满足,那么本步骤将直接被跳过,进入到下一个步骤.
  4. 间隔时间指本步骤处于结束状态后,必须等待的时间.只有间隔时间被等待后,流程才能进入到下一个步骤.
  5. 超时时间是指步骤在匹配执行或者结束条件未成功的情况下,最多等待的时间长度.如果在超时时间以内,没有匹配到执行或者结束条件,那么认为本步骤已经超时,流程将重新从第一个步骤或者任意步骤尝试匹配,然后从已匹配步骤重新运行流程实例.

执行条件​

执行条件设定了步骤满足什么条件才可以执行动作,步骤只有在执行条件被匹配到的时候,动作才会被执行.

条件​

条件由名称,匹配对象,置信度,排除匹配对象组成.

  1. 名称即条件的名称
  2. 匹配对象即如果出现了匹配对象那么条件为真.
  3. 置信度,某一些对象出现了,但是系统也无法完全确认是该对象,需要一个置信度阈值,来辅助确定.例如系统判定某一对象80%的几率是对象A,如果对象A的置信度是0.7,那么条件为真,如对象A的置信度为0.9,那么条件为假,
  4. 排除匹配对象表示如果对象被匹配到,那么条件为假,如果没有被匹配到,那么条件为真.

执行动作​

执行动作设定了步骤的执行动作.执行动作如果可以被重复执行,每一次都需要匹配执行条件

动作​

动作由名称,参数,操作类型,操作对象构成.

  1. 名称即动作名称
  2. 参数指动作执行过程中需要用到的数据.
  3. 操作类型包括:点击,双击,长按,滑动,键盘输入,文本框输入,执行脚本,步骤跳转.
  4. 操作对象指动作执行的目标对象,并不是所有的动作都需要在目标对象上执行.

操作类型说明​

  1. 点击,双击,长按都是单操作目标.
  2. 滑动需要多个操作目标,滑动的顺序与操作对象加入的先后有关.
  3. 键盘输入实现将参数作为脚本执行后的结果输入到当前激活的输入框中.
  4. 文本框输入实现将参数作为脚本执行后的结果输入到当前匹配到的文本框空间节点中.
  5. 执行脚本实现将参数作为脚本进行执行.
  6. 步骤跳转,跳转到参数(步骤名称)指定的步骤.

结束条件​

结束条件设定了步骤满足什么条件才可以结束.步骤只有在结束条件被匹配到的时候,步骤才会由完成状态进入到结束状态.
